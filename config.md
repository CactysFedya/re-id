# Конфигурация `pipeline.toml`

Модуль можно запускать без пользовательского конфига: встроенные defaults загружаются автоматически.

Если требуется настройка, пользователь может передать частичный `TOML`. В таком файле допускается указывать только те параметры, которые нужно изменить.

## Быстрые сценарии

Полный шаблон:

```powershell
reid-dump-config
```

Сохранить шаблон в файл:

```powershell
reid-dump-config configs/pipeline.user.toml
```

Запуск без конфига:

```powershell
reid-run --video assets/mot17/mot17-05-sdp-001-207.avi
```

Запуск с частичным конфигом:

```powershell
reid-run --config configs/pipeline.user.toml
```

## Минимальный пример частичного конфига

```toml
[reid.source]
type = "rtsp"
uri = "rtsp://127.0.0.1:8554/stream"

[reid.detector]
model_name = "yolo26n.pt"
conf = 0.2

[reid.extractor]
model_name = "osnet_x1_0"
weights_path = "models/reid/custom_osnet.pth"
```

## Доступные параметры

### `[reid]`

- `outputs_root`: каталог для артефактов запуска. По умолчанию `outputs`.
- `run_prefix`: префикс подпапки запуска. По умолчанию `reid`.
- `output_video_name`: имя итогового видеофайла. По умолчанию `demo_reid.mp4`.
- `log_every`: период логирования по кадрам. Положительное целое.
- `metrics_file_name`: имя файла метрик. По умолчанию `metrics.json`.

### `[reid.source]`

- `type`: один из `file`, `usb`, `camera`, `rtsp`, `http`, `https`, `mjpeg`.
- `uri`: путь к видео или URI потока.
- `device_index`: индекс камеры для `usb`/`camera`.
- `reconnect`: переподключаться ли к live-источнику.
- `reconnect_delay_s`: задержка между попытками переподключения.
- `reconnect_max_attempts`: число попыток переподключения.

### `[reid.output]`

- `save_video`: сохранять ли размеченное видео.
- `save_assignments`: сохранять ли `assignments.csv`.

### `[reid.stop]`

- `max_frames`: остановка после заданного числа кадров. Пустая строка отключает лимит.
- `max_duration_s`: остановка по времени работы в секундах. Пустая строка отключает лимит.

### `[reid.detector]`

- `model_name`: имя модели Ultralytics.
- `weights_path`: путь к локальным весам детектора.
- `conf`: порог уверенности детектора, диапазон `0..1`.
- `classes`: список классов детектора, например `[0]`.

### `[reid.extractor]`

- `model_name`: имя модели Torchreid.
- `weights_path`: путь к пользовательским весам экстрактора.
- `device`: `cpu`, `cuda`, `cuda:0` или пустое значение для автоопределения.

### `[reid.gallery]`

- `sim_threshold`: порог сопоставления с галереей, диапазон `0..1`.
- `ema`: коэффициент обновления прототипа, диапазон `0..1`.
- `update_threshold`: порог обновления существующей идентичности, диапазон `0..1`.
- `max_ids`: максимальный размер галереи.
- `state_path`: путь к файлу состояния галереи.
- `load_on_start`: загружать ли галерею перед запуском.
- `save_on_exit`: сохранять ли галерею после завершения.
- `autosave_interval_s`: интервал автосохранения галереи в секундах.

### `[reid.tracker]`

- `iou_threshold`: порог IoU для локального трекера, диапазон `0..1`.
- `max_missed`: сколько кадров трек может пропустить.
- `confirm_hits`: число подтверждений перед назначением глобального `id`.

## Правила путей

- Если указан `--config`, все относительные пути внутри `TOML` считаются относительно папки этого файла.
- Если `--config` не указан, относительные пути считаются относительно текущей рабочей директории или `--work-dir`.
- Для `weights_path`, `state_path`, `outputs_root` и файлового `source.uri` можно использовать как относительные, так и абсолютные пути.
