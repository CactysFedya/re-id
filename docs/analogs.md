# Сравнение аналогов

## Принцип отбора аналогов

Поиск производился среди статей и научных трудов с помощью таких платформ, как: `Google Scholar`, `eLibrary`, `arXiv`, `CyberLeninka`, `IEEE Xplore`, `MDPI`. Для поиска использовались следующие запросы и ключевые слова: `re-id`, `re-identification`, `person re-identification` `re-identification in dark`, `re-identification night time`, `re-identification infrared`, `re-identification thermal`, `visible-infrared person re-id`, `cross-modality person re-id`. Основным критериями отбора являлись:

- источники 2025 года;
- принадлежность метода к задаче кросс-модальной идентификации (RGB-IR);
- наличие открытой реализации (OpenSource) или подробного архитектурного описания
- наличие опубликованных результатов тестирования на общепризнанном бенчмарке SYSU-MM01.

### AGPI2 (Adaptive Generation of Privileged Intermediate Information)

Данный метод реализует подход к обучению, при котором генерируется виртуальное «привилегированное» промежуточное представление, служащее мостом между видимой и инфракрасной модальностями. Ключевой особенностью является использование генеративного модуля (GAN) только на этапе обучения. Это позволяет трансформировать признаки видимого спектра в пространство, близкое к инфракрасному, без использования генератора на этапе тестирования, что приводит к сложному и долгому обучению, но повышает точность при тестировании, не замедляя работу системы.[1]

### FSRL (Fine-Grained Semantic Representation Learning)

Метод FSRL фокусируется на извлечении среднеуровневых признаков, содержащих детальную семантическую информацию об идентификации (детали одежды, форма тела). Архитектура включает модули нормализации и компенсации модальностей (MNC), агрегации промежуточных признаков (IFA) и семантически согласованного обучения (SAFL). Подход направлен на повышение точности за счет использования устойчивых к смене модальности деталей, но требует большего количества параметров.[2]

### CECR-Net (Channel Enhanced Cross-modality Relation Network)

Метод реализует двухпоточную архитектуру, нацеленную на захват структурных отношений между частями тела, инвариантных к модальности (эффективно используется структурная информация тела, которая не меняется с изменением модальности, позы или освещения). Метод использует стратегию Channel-Enhanced Sharing (CES), где видимые изображения случайным образом преобразуются для имитации одноканальных ИК-изображений, что повышает устойчивость к вариациям цвета. Архитектура включает модули Intra-Relation Learning (IRL) и Cross-Relation Learning (CRL), основанные на реляционных сетях (relation network), для моделирования структурных связей внутри и между модальностями соответственно.[3]

### CSDN (CLIP-Driven Semantic Discovery Network)

CSDN использует предварительно обученную модель CLIP (Language-Image Pre-training) для интеграции высокоуровневой семантики (пол, атрибуты одежды) в визуальные признаки. Особенности реализации: Modality-specific Prompt Learner (MsPL) для генерации бимодальных (отдельно для RGB и IR) языковых описаний, Semantic Information Integration (SII) с механизмом внимания для объединения семантических деталей из V и I описаний, High-level Semantic Embedding (HSE) для встраивания богатых текстовых признаков в визуальные представления. Все указанные дополнительные модули используются только на этапе обучения, что сохраняет высокую скорость работы.[4]

### MTRL (Modality-Transition Representation Learning)

Фреймворк MTRL использует сгенерированные промежуточные изображения как «передатчик» для выравнивания признаков видимого и инфракрасного спектров. Главная архитектурная особенность — отсутствие дополнительных параметров на этапе вывода. Улучшение качества достигается за счет использования специфических функций потерь (Modality-Aware Hierarchical Constraints Loss и Modality-Query Regularization Loss), которые  основываются на сгенерированных изображениях, уменьшая различия между модальностями.[5] Использование генератора добавляет 16% ко времени обучения в сравнении с baseline методом, в качестве которого в исследовании используется SAAI.[6]

## Критерии сравнения аналогов

### Механизм выравнивания модальностей

Данный критерий отображает архитектурную особенность построения метода, направленную на уменьшение разрыва между признаками RGB и IR изображений. Это качественный критерий, определяющий, за счет чего достигается инвариантность: генерация промежуточных данных (GAN), использование семантических подсказок, структурный анализ частей тела или аугментация каналов. Выбор механизма напрямую влияет на сложность обучения и устойчивость к шуму.

### Rank-1

Критерий Rank-1 показывает вероятность того, что верное изображение искомого человека окажется первым среди выборки по запросу.

Формула расчета Rank-1:

Пусть есть $ Q $ запросов.
Для каждого запроса $ q $ система выполняет поиск по галерее и возвращает ранжированный список результатов, где 1-я позиция — самый похожий объект.

Проверяется, является ли результат на **первой позиции** релевантным (правильным):

$$
\text{Rank-1}(q) = rel(1)
$$

где $ rel(k) $ — функция релевантности результата на k-й позиции:

Итоговая метрика **Rank-1 Accuracy** вычисляется как среднее по всем запросам:

$$
\text{Rank-1} = \frac{1}{Q} \sum_{q=1}^{Q} \text{Rank-1}(q) = \frac{\text{Число запросов с правильным 1-м результатом}}{\text{Общее число запросов } Q}
$$

Аналогичным образом для тестирования систем re-id при обучении зачастую вычисляются критерии $ Rank-N, N \in \{5, 10, 20\} $, но самым важным из них является Rank-1, так как он наиболее применим на практике.

### mAP

Метрика mAP (mean Average Precision) в задаче повторной идентификации отражает среднюю точность ранжирования по множеству запросов.

Формула расчета mAP:

Пусть есть $ Q $ запросов.
Для каждого запроса $ q $ вычисляется **Average Precision (AP)**:

$$
AP(q) = \frac{1}{K_q} \sum_{k=1}^{N} P(k) \cdot rel(k)
$$

где:

- $ N $ — общее число изображений в галерее,
- $ rel(k) = \begin{cases} 1, & \text{если } k\text{-й результат релевантен} \\ 0, & \text{иначе} \end{cases} $
- $ P(k) = \frac{\text{число релевантных результатов среди первых } k}{k} $
- $ K_q $ — общее число релевантных изображений для запроса $ q $ в галерее.

Затем mAP вычисляется как среднее по всем запросам:

$$
mAP = \frac{1}{Q} \sum_{q=1}^{Q} AP(q)
$$

По mAP можно оценить общую способность алгоритма находить все релевантные результаты идентификации объекта и возвращать их на высокие позиции результатов поиска.

## Выводы по итогам сравнения

На основе проведенного анализа можно сделать следующие выводы:

Методы **AGPI2** и **MTRL** демонстрируют наиболее эффективный подход для реальных систем видеонаблюдения. Они используют сложные вычисления (генерацию изображений) только на этапе обучения, что позволяет достигать высокой точности, не замедляя работу системы на этапе вывода.

Метод **CSDN**, использующий мультимодальную модель CLIP, показывает перспективность интеграции текстовых описаний. Это дает высокую точность, однако качество работы зависит от предобученной модели CLIP и может быть ограничено "грубостью" семантических токенов.

Метод **FSRL** показывает высокие результаты по метрикfv точности за счет учета мелких деталей, но платит за это увеличением числа параметров (с 56M до 76M) и снижением FPS [2], что может быть критично для встраиваемых систем.

**CECR-Net** предлагает подход к аугментации данных без использования GAN, что снижает вычислительные затраты на обучение, а также дает довольно низкое время отлика (2.6 мс [3]) но демонстрирует чуть более низкие показатели точности по сравнению с другими аналогами.

Для задач, требующих работы в реальном времени с ограниченными ресурсами, наиболее предпочтительными архитектурами являются те, которые выносят вычислительную сложность на этап обучения (AGPI2, MTRL).

## Выбор метода решения

Реализуемый программный модуль должен представлять из себя полноценный, стандартный для применяемого на практике re-id модуля, пайплайн (конвейер), включающий детекцию, выделение признаков и сопоставление с галереей ранее идентифицированных объектов. В данном исследовании объектами являются люди в кадрах видеозаписи.

Для детекции будет применена модель YOLO, зарекомендовавшая себя как прооверенный временем и постоянно обновлямый фреймворк для детекции и трекинга объектов [9].

Решение задачи re-id:
За основу метода идентификации будет взят один из методов, использующих генеративный подход: **AGPI2** или **MTRL** с модификацией процедуры обучения с помощью добавления аугментации данных или объединения подходов, используемых в данных методах.

Для подтверждения эффективности выбранного метода необходимо провести следующие эксперименты:

- Тестирование на стандартном датасете SYSU-MM01 по критериям точности (значение Rank-1, mAP) в режиме All Search. Ожидаются значения Rank-1 > 76%, mAP > 75%.
- Замер времени отклика полученного программного модуля для оценки применимости в системах видеонаблюдения реального времени. Ожидается сохранение скорости на уровне baseline методов, не использующих генеративный подход для обучения.
- Визуализация распределения признаков (t-SNE) для видимых и инфракрасных изображений одной персоны до и после применения предложенного метода обучения. Ожидается более плотная кластеризация признаков одной личности независимо от спектра.
