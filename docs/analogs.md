# Сравнение аналогов

## Принцип отбора аналогов

Поиск производился среди статей и научных трудов с помощью таких платформ, как: `Google Scholar`, `eLibrary`, `arXiv`, `CyberLeninka`, `IEEE Xplore`, `MDPI`. Для поиска использовались следующие запросы и ключевые слова: `re-id`, `re-identification`, `person re-identification` `re-identification in dark`, `re-identification night time`, `re-identification infrared`, `re-identification thermal`, `visible-infrared person re-id`, `cross-modality person re-id`. Основным критерием отбора являлась принадлежность метода к задаче кросс-модальной идентификации (RGB-IR) и наличие опубликованных результатов тестирования на общепризнанном бенчмарке SYSU-MM01. Исследуемые методы можно условно разделить по конкретному подходу к решению проблемы: методы на основе внимания, генеративные методы и методы, основанные на аугментации или изменении обучающих данных.

### AGW (Attention Generalized along with Weighted Regularization) [1]

Базовый метод повторной идентификации человека, принятый как стандарт для сравнения с новыми алгоритмами (например в [4]). AGW использует двухпоточную архитектуру ResNet-50 с добавлением блоков обобщенного внимания для извлечения глобальных признаков, что позволяет улучшить точность в условиях кросс-модальной идентификации. Метод не использует специальные техники аугментации данных, а направлен на усовершенствование архитектуры сети.

### DDAG (Dynamic Dual-Attentive Aggregation Learning) [2]

Данный метод направлен на работу с локальными признаками - конкретно на выделение частей тела человека. DDAG использует механизм динамического двойного внимания (что отражено в названии метода) для определения "важности" различных частей тела человека. Архитектура метода основана на построении внутримодальных и межмодальных графовых связей, чтобы сопоставить части тела в условиях зашумленных данных (например, если лицо закрыто - вес связанного параметра снижается). Обработка графов и механизмов внимания добавляет вычислительной сложности, что затрудняет внедрение метода в реальные условия.

### JSIA-ReID (Joint Set-Level and Instance-Level Alignment) [3]

Метод, представляющий генеративный подход - использует генеративно состязательные сети (GAN) для создания синтетических изображений (из RGB генерируются IR и наоборот), таким образом пытаясь свести задачу к одномодальной. Основным недостатком генеративного подхода является необходимость необходимость обучения генератора и дискриминатора[1], что сильно увеличивает время тренировки и требует больших вычислительных мощностей - это в свою очередь делает метод трудноприменимым в системах идентификации в реальном времени.

### MSCLNet (Modality Synergy Complement Learning) [4]

Данный метод направлен на решение проблемы потери специфической информации при объединении признаков разных модальностей. MSCLNet использует каскадную стратегию агрегации: то есть признаки RGB и IR объединяются постепенно на разных уровнях глубины сети. Это позволяет методу достигать довольно высокой точности в сравнении с другими, но при этом из-за использования каскадной структуры на обработку каждого кадра требуется больше времени в сравнении с одноступенчатыми методами (в которых агрегация признаков происходит один раз либо на выходе сети, либо до подачи в сеть)

### HAT (Homogeneous Augmented Tri-Modal Learning) [5]

Метод, использующий аугментацию данных. HAT преобразует RGB-изображения в оттенки серого, создавая таким образом третью, промежуточную модальность, которая направлена на улучшение обобщающей способности метода за счет снижения расстояния между RGB и IR в пространстве признаков. Но из-за такой структуры метод использует трехпоточную сеть (для обработки каждого цветового канала), что усложняет архитектуру и увеличивает время обучения.

## Критерии сравнения аналогов

### Rank-1

Критерий Rank-1 показывает вероятность того, что верное изображение искомого человека окажется первым среди выборки по запросу.

Формула расчета Rank-1:

Пусть у нас есть $ Q $ запросов.
Для каждого запроса $ q $ система выполняет поиск по галерее и возвращает ранжированный список результатов, где 1-я позиция — самый похожий объект.

Проверяется, является ли результат на **первой позиции** релевантным (правильным):

$$
\text{Rank-1}(q) = rel(1)
$$

где $ rel(k) $ — функция релевантности результата на k-й позиции:

Итоговая метрика **Rank-1 Accuracy** вычисляется как среднее по всем запросам:

$$
\text{Rank-1} = \frac{1}{Q} \sum_{q=1}^{Q} \text{Rank-1}(q) = \frac{\text{Число запросов с правильным 1-м результатом}}{\text{Общее число запросов } Q}
$$

Аналогичным образом для тестирования систем re-id при обучении зачастую вычисляются критерии $ Rank-N, N \in \{5, 10, 20\} $, но самым важным из них является Rank-1, так как он наиболее применим на практике.

### mAP

Метрика mAP (mean Average Precision) в задаче повторной идентификации отражает среднюю точность ранжирования по множеству запросов.

Формула расчета mAP:

Пусть у нас есть $ Q $ запросов.
Для каждого запроса $ q $ вычисляется **Average Precision (AP)**:

$$
AP(q) = \frac{1}{K_q} \sum_{k=1}^{N} P(k) \cdot rel(k)
$$

где:

- $ N $ — общее число изображений в галерее,
- $ rel(k) = \begin{cases} 1, & \text{если } k\text{-й результат релевантен} \\ 0, & \text{иначе} \end{cases} $
- $ P(k) = \frac{\text{число релевантных результатов среди первых } k}{k} $
- $ K_q $ — общее число релевантных изображений для запроса $ q $ в галерее.

Затем mAP вычисляется как среднее по всем запросам:

$$
mAP = \frac{1}{Q} \sum_{q=1}^{Q} AP(q)
$$

По mAP можно оценить общую способность алгоритма находить все релевантные результаты идентификации объекта и возвращать их на высокие позиции результатов поиска.

### mINP

Метрика mINP (mean Inverse Negative Penalty) в задаче повторной идентификации показывает, сложность нахождения всех релевантных результатов. Была введена в [1] вместе с методом AGW.

Формула расчета mINP:

### Формула расчета mINP

Пусть у нас есть $ Q $ запросов.

Для каждого запроса $ q $ вычисляется **Inverse Negative Penalty (INP)**.

Сначала найдем позицию последнего релевантного результата:

$$
P_{\text{worst}}(q) = \max\{k \mid rel(k) = 1\}
$$
где $ rel(k) $ — релевантность $ k $-го результата (1 — релевантен, 0 — нет).

Затем для каждого запроса вычисляется:
$$
INP(q) = \frac{R_q}{R_q - P_{\text{worst}}(q) + 1}
$$
где $ R_q $ — общее число релевантных изображений для запроса $ q $ в галерее.

Наконец, **mINP** вычисляется как среднее по всем запросам:
$$
mINP = \frac{1}{Q} \sum_{q=1}^{Q} INP(q)
$$

По mINP можно оценить, насколько система удобна в реальном применении, оценивая дополнительные усилия для нахождения всех правильных ответов.

## Выводы по итогам сравнения

По результатам сравнения можно заметить, что лучшие результаты среди сравниваемых аналогов имеет MSCLNet, превышая другие методы по всем используемым метрикам. Следовательно этот метод будет взят за основу программного модуля.

JSIA-ReId наоборот показал худшие результаты, что объясняется тем, что этот аналог самый ранний из всех представленных (был предлставлен в 2020 году, еще до появления базового метода AGW и введения метрики mINP). По результату можно судить, что применение GAN не является хорошим решением для поставленной задачи.

Подходы, применяемые методами DDAG и HAT являются более уместными, так как имеют прирост по всем метрикам в сравнении с базовым AGW. Как видно из таблицы, DDAG, использующий подход с механизмами внимания, дает лучший прирост в mINP, следовательно позволяет эффективнее находить сложные примеры в запросах. HAT, использующий подход аугментации данных, в свою очередь значительнее увеличивает значения метрик Rank-1 и mAP, что означает упор на повышение общей точности идентификации.

В итоге метод HAT можно поставить на второе место в условном общем списке аналогов после MSCLNet, что говорит о том, что добавление аугментации данных при обучении метода re-id может улучшить метрики и дать более точные результаты.

## Выбор метода решения

Реализуемый программный модуль должен представлять из себя полноценный, стандартный для применяемого на практике re-id модуля, пайплайн (конвейер), включающий детекцию, выделение признаков и сопоставление с галереей ранее идентифицированных объектов. В данном исследовании объектами являются люди в кадрах видеозаписи.

Для детекции будет применена модель YOLO, зарекомендовавшая себя как надежный и постоянно обновлямый фреймворк для детекции и трекинга объектов [8].

За основу метода идентификации будет взят подход каскадной агрегации, применяемой в методе MSCLNet, с модификацией процедуры обучения с помощью добавления аугментации данных.

Для оценки качества работы модуля модифицированный метод re-id будет протестирован на стандартном датасете SYSU-MM01 по таким же критериям (значение Rank-1, mAP, и mINP).
